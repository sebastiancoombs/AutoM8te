# 2026-04-02 — Voice Bridge Built & Tested

## What Happened
Built the Discord voice → OpenAI Realtime API → drone control bridge (`voice_bridge/`).

## Key Decisions
- **OpenAI Realtime API (beta format)**: The GA API format (with nested `audio.input/output`) doesn't work on the `gpt-4o-realtime-preview` model yet. Reverted to beta format: `input_audio_format: 'pcm16'`, `output_audio_format: 'pcm16'`, `voice`, `turn_detection: { type: 'server_vad' }`. This works.
- **Voice: ash** — military copilot vibe
- **10 tools registered**: takeoff, land, hover, rtl, emergency, move, formation, search, status, group_assign
- **Intent layer proxy**: voice bridge calls `POST http://localhost:8080/api/tool` on the intent layer. Keeps voice bridge thin.
- **Audio pipeline**: Discord Opus 48kHz → PCM mono → downsample 2:1 to 24kHz → base64 → Realtime API. Response: PCM 24kHz → upsample 2:1 to 48kHz → Opus encode → Discord playback.
